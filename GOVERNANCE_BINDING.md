## Repository Enforcement Disclosure

This repository enforces governance through platform-level controls,
not documentation alone.

The following controls are active:

- Protected default branch (`main`)
- Pull-request-only changes
- Required approving reviews
- CODEOWNERS enforcement
- Verified (signed) commits
- Canonical lock tags

Direct modification of protected branches is intentionally disallowed.

These controls bind human authority, governance standards, and licensing
artifacts into a non-bypassable enforcement chain.
